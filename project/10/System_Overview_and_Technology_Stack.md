# Проект: Система рекомендаций в игровой индустрии

## 1. Цель проекта

Разработка клиент-серверного веб-приложения, предоставляющего пользователям персонализированные рекомендации игр на основе их предпочтений и истории оценок, а также инструменты публикации и модерации контента.

Проект реализуется в архитектуре REST API с использованием контейнеризации (Docker).

# 2. Функционал для реализации (MVP)

## 2.1 Роль: Игрок

### 1. Регистрация и аутентификация
- Регистрация пользователя (username, email, password)
- Авторизация пользователя
- Выход из системы
- Хранение токена аутентификации на клиенте

### 2. Профиль пользователя
- Просмотр информации профиля:
  - Никнейм
  - Email
  - Дата регистрации
  - Количество оценённых игр
  - Количество написанных отзывов
- Возможность изменения пароля
- Возможность удаления аккаунта

### 3. Управление предпочтениями
- Выбор любимых жанров
- Просмотр выбранных жанров
- Обновление предпочтений

### 4. Просмотр игр
- Страница **Explore** с рекомендованными играми
- Просмотр списка игр
- Переход на страницу конкретной игры

### 5. Страница игры
- Название
- Описание
- Системные требования
- Жанры
- Средняя оценка
- Список отзывов пользователей

### 6. Оценки и отзывы
- Возможность поставить оценку (1–5)
- Возможность оставить один отзыв к игре
- Возможность изменить оценку
- Автоматическое обновление статистики игры

### 7. Рекомендательная система
В системе используется алгоритм контентной фильтрации (Content-Based Filtering) на основе жанров.

- Если у пользователя отсутствует история взаимодействия, отображаются популярные игры, отсортированные по средней оценке.
- При наличии предпочтений для каждой игры вычисляется рекомендационный балл как сумма весов совпадающих жанров пользователя.
- Итоговый список формируется сортировкой игр по убыванию рассчитанного балла с исключением уже оценённых игр.

## 2.2 Роль: Разработчик

- Регистрация с ролью разработчика
- Добавление новой игры:
  - Название
  - Описание
  - Жанры
  - Системные требования
- Просмотр списка своих игр
- Просмотр агрегированной статистики:
  - Средняя оценка
  - Количество отзывов

## 2.3 Роль: Модератор (менеджер платформы)

- Просмотр новых добавленных игр
- Одобрение или отклонение игр
- Просмотр отзывов пользователей
- Изменение статуса модерации отзывов
- Возможность временной блокировки пользователей

## 2.4 Логирование активности

Система фиксирует:
- Авторизацию пользователя
- Просмотр игры
- Выставление оценки
- Добавление отзыва

Логи используются для аналитики и демонстрации работы системы.

# 3. Архитектура системы

Система реализуется по трёхуровневой архитектуре:

- Клиентский уровень (Frontend)
- Сервер приложений (Backend)
- База данных (Database)

Взаимодействие осуществляется через REST API.

# 4. Стек технологий

## Frontend
- HTML
- CSS
- JavaScript
- Nginx

## Backend
- Python
- FastAPI
- SQLAlchemy (ORM)
- JWT (аутентификация)

## Database
- PostgreSQL

## Инфраструктура
- Docker
- Docker Compose

# 5. Развёртывание

Проект разворачивается с использованием Docker Compose и включает:

- Контейнер Nginx (веб-сервер и reverse proxy)
- Контейнер Backend (FastAPI-приложение)
- Контейнер PostgreSQL (база данных)

Все компоненты взаимодействуют внутри одной Docker-сети.

# 6. Результат реализации

В результате будет создано полнофункциональное клиент-серверное веб-приложение, демонстрирующее:

- Работу REST API
- Аутентификацию и разграничение ролей
- Работу с реляционной базой данных
- Контейнеризацию и инфраструктуру Docker
- Реализацию базовой рекомендательной системы
